# Plan: Fix White Image Issue with createImageBitmap

## Problem
Canvas-based compression produces white/black images for certain formats (CMYK, HEIC, EXIF rotation issues). The current `loadImage` uses `new Image()` which doesn't handle color spaces or EXIF orientation properly.

## Solution
Replace `new Image()` + `drawImage()` with `createImageBitmap()` which has:
- **`imageOrientation: 'from-image'`** — handles EXIF rotation automatically
- **`colorSpaceConversion: 'default'`** — proper CMYK/sRGB conversion
- **Built-in resize** — `resizeWidth`/`resizeHeight` options avoid extra canvas operations

## Changes — `src/lib/compressImage.ts`

Replace the `loadImage` + canvas approach with:

```typescript
async function loadImage(file: File): Promise<ImageBitmap> {
  return createImageBitmap(file, {
    imageOrientation: 'from-image',    // fix EXIF rotation
    colorSpaceConversion: 'default',   // fix CMYK/sRGB
    resizeWidth: targetWidth,           // optional: resize during decode
    resizeHeight: targetHeight,
    resizeQuality: 'high',
  })
}
```

Then draw the bitmap to canvas and export as WebP.
